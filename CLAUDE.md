## Commands

```bash
npm run dev        # Dev server at http://localhost:4321
npm run build      # Production build to dist/
npm run preview    # Preview production build locally
```

### AI Blog Publishing

```bash
# Requires: pip install anthropic python-slugify
# Requires: ANTHROPIC_API_KEY env var

python scripts/publish_post.py "Topic Name" --category "Category"   # Specific topic
python scripts/publish_post.py --auto                                # Auto-pick next unpublished topic
python scripts/publish_post.py "Topic" --category "Cat" --dry-run   # Preview without saving
python scripts/publish_post.py "Topic" --category "Cat" --no-push   # Save but don't git push
```

## Architecture

Astro 5 static site for Body Nutrition (contract supplement manufacturer in St. Petersburg, FL). Deploys to Netlify via git push — `netlify.toml` configures the build.

### Page Routing

- **Corporate pages** (`src/pages/index.astro`, `capabilities.astro`, `contact.astro`): Static `.astro` files, edited directly.
- **Blog**: Content-driven via markdown files in `src/content/blog/`. `blog/index.astro` auto-lists all posts via `Astro.glob()`. `blog/[slug].astro` generates individual pages using `getStaticPaths()`.

### Layouts & Components

- `BaseLayout.astro` wraps every page (HTML head, Header, Footer). Accepts `title` and optional `description` props.
- `BlogPostLayout.astro` wraps individual blog posts (hero + content + sidebar).
- Components: `Header.astro`, `Footer.astro`, `BlogCard.astro`.

### Blog Post Format

Markdown files in `src/content/blog/` with this frontmatter:

```yaml
title: "Post Title"
slug: "url-slug"
category: "Category Name"
emoji: "🌿"
excerpt: "Meta description text"
date: "YYYY-MM-DD"
```

Posts are generated by `scripts/publish_post.py` which calls the Claude API, writes the markdown file, and optionally commits/pushes.

### Styling

- CSS variables defined in `src/styles/global.css` (brand palette, fonts, layout).
- Fonts: Instrument Serif (display), DM Sans (body) — loaded via Google Fonts in BaseLayout.
- Component styles use Astro scoped `<style>` blocks.
- Brand colors: `--navy` (dark), `--teal` (primary accent), `--gold` (secondary accent).
